Create download_model.py that downloads the Nemotron GGUF to the network volume. Requirements:

**Model details:**
- Repo: `nvidia/Nemotron-3-Super-120B-Instruct` (or the unsloth GGUF repo — use `unsloth/Nemotron-3-Super-120B-Instruct-GGUF`)
- File pattern: `*UD-Q4_K_XL*` (use `allow_patterns` to download only this quant, not the entire repo)
- Destination: `/workspace/models` — must match the path in nemotron.py

**HF_TOKEN handling:**
- Read from `os.environ.get("HF_TOKEN")` — never hardcode
- If not set, print a clear error and exit with a non-zero code: "Error: HF_TOKEN environment variable not set. Get your token from https://huggingface.co/settings/tokens"

**Script structure:**
```python
#!/usr/bin/env python3
"""
Download Nemotron-3-Super-120B-Instruct (UD-Q4_K_XL) to the RunPod network volume.

Run this once to seed the volume before first deploy:
    HF_TOKEN=hf_... python download_model.py

The model (~60 GB) must be present at /workspace/models before the endpoint
starts, otherwise the cold-start will timeout trying to download during inference.
"""
```

Use `huggingface_hub.snapshot_download` with:
- `repo_id` — the GGUF repo
- `allow_patterns=["*UD-Q4_K_XL*"]` — download only the target quant file
- `local_dir="/workspace/models"` — destination matching nemotron.py
- `token=hf_token` — the env var value

Add a print statement showing download progress context:
- Before: print the repo, pattern, and destination so the user knows what's happening
- After: print the path of the downloaded file(s) and a reminder of the next step ("Now run: flash deploy nemotron.py")

No argparse needed — keep it simple. The token comes from env, path is fixed.
