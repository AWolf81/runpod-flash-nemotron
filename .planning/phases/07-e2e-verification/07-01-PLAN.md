Structure the doc as follows:

## Overview
Brief statement: this checklist verifies the full deploy flow from a blank volume. Note: must use a fresh/clean volume — re-running on a volume that already has the binary and model cached will NOT test the cold start path and will mask any initialization bugs.

## Prerequisites
- RunPod account with Flash access and credits
- HuggingFace token with access to `unsloth/NVIDIA-Nemotron-3-Super-120B-A12B-GGUF`
- Python 3.11+ with `runpod-flash` installed
- A **fresh** RunPod network volume (no existing `/runpod-volume/cache/` or model files)

## Verification Steps

Number each step 1–9. For each step, include:
- The exact command to run
- Expected output / what to look for
- A "Result: [ ] Pass / [ ] Fail / Notes: ___" field
- A "Time: ___" field where timing matters

Steps:
1. **Export credentials** — set RUNPOD_API_KEY and HF_TOKEN env vars (exact export commands)
2. **Seed the volume** — `HF_TOKEN=... python nemotron.py seed` — expect llama-server build (~5–10 min) + model download (~84 GB, ~20–30 min total first time). Record total time.
3. **Verify volume contents** — check that `/runpod-volume/cache/llama-server` exists and model GGUF shards are present. Commands: use the `/admin/install` endpoint (after deploy) OR SSH into a worker. Note: include the SSH path from README (console.runpod.io → Serverless → nemotron → Workers → Running worker → Connect). Alternative: `flash run` then `curl /health` → look for `missing_binary` or `missing_model` status.
4. **Deploy** — `flash deploy --env production` (or `flash run` for dev). Record deployed endpoint URL.
5. **Poll /health until cold** — `curl https://<endpoint>/health` → expect `{"status":"cold"}` on first hit (worker up but warmup not triggered).
6. **Run warmup** — `time bash warmup.sh <endpoint-id>` — this scales the worker, triggers `/warmup`, polls /health, and waits for `ready`. Record elapsed time.
7. **Send test inference** — exact curl command:
   ```bash
   curl https://<endpoint-id>.api.runpod.ai/v1/chat/completions \
     -H "Content-Type: application/json" \
     -H "Authorization: Bearer $RUNPOD_API_KEY" \
     -d '{"model":"nemotron","messages":[{"role":"user","content":"Say hello in one sentence."}],"max_tokens":50}'
   ```
   Expect: JSON response with `choices[0].message.content` containing a greeting. Record time to first token.
8. **Send second inference** (verify slot priming worked) — same curl. Expect: second request responds immediately (no timeout). Record latency.
9. **Scale down** — Ctrl+C in warmup.sh terminal OR `curl -X POST .../scale-down`. Verify endpoint returns to 0 workers.
