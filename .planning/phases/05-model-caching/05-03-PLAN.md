<task type="checkpoint:human-action" gate="blocking">
  <action>Configure RunPod cached model for the nemotron-super-120b endpoint in the RunPod console</action>
  <instructions>
The Flash SDK has no API for cached model configuration — it must be set via the RunPod console.

1. Go to: RunPod Console → Serverless → Endpoints → `nemotron-super-120b`
2. Click **Settings** (or Edit)
3. Find the **Model** section (it may be labeled "Hugging Face Model" or "Cached Model")
4. Set:
   - **Hugging Face Model ID**: `[your private repo ID — e.g. your-org/nemotron-q4-xl]`
   - **Hugging Face API Key**: your HF_TOKEN (needs read access to the private repo)
5. **Save** the endpoint settings

RunPod will begin pre-caching the model files on fleet hosts in the background. This takes time (it's downloading ~83.8 GB), but you are **not billed** for this caching time.

You do NOT need to redeploy. The cached model setting takes effect on the next worker cold start.
  </instructions>
  <verification>RunPod console shows the endpoint has a cached model configured (Hugging Face Model ID is populated)</verification>
  <resume-signal>Type "configured" when the endpoint settings are saved with the cached model ID</resume-signal>
</task>
