# Requirements: runpod-flash-nemotron

**Defined:** 2026-03-20
**Core Value:** Give any developer a self-hosted AI coding assistant at Claude Code / Codex quality and speed, paying only for GPU seconds used on RunPod serverless — not a subscription.

## v1 Requirements

### Deployment

- [x] **DEPL-01**: Developer can deploy a serverless endpoint with a single `flash deploy` command using `nemotron.py`
- [x] **DEPL-02**: Deployment script configures A100 80GB GPU (`GpuGroup.AMPERE_80`), 100 GB Network Volume, FlashBoot, and `execution_timeout=1800` — *Note: actual hardware is RTX Pro 6000 Blackwell 96GB; A100 OOMs*
- [x] **DEPL-03**: Developer can run a one-time remote seed job using `python nemotron.py seed` before first deploy, preventing cold-start timeout loop
- [x] **DEPL-04**: llama-server starts with correct flags (`-ngl 99 --override-tensor "exps=CPU" -c 32768 -fa --no-mmap -np 1 --cont-batching`), exposing `/v1/chat/completions` on port 8080
- [x] **DEPL-05**: `execution_timeout=1800` is set with inline comment explaining why
- [x] **DEPL-06**: `--override-tensor "exps=CPU"` is documented with explanation that MoE expert weights exceed VRAM and must route to CPU RAM
- [x] **DEPL-07**: `idle_timeout` tuning is documented with guidance on reducing billing for short coding sessions

### Integration

- [x] **INTG-01**: Developer can configure Claude Code to use the RunPod endpoint via LiteLLM gateway — *Note: direct OpenAI endpoint not compatible; Claude Code expects Anthropic-format traffic*
- [x] **INTG-02**: Developer can configure OpenCode to use the RunPod endpoint via `~/.config/opencode/config.json`
- [x] **INTG-03**: Developer can configure Mistral Vibe to use the RunPod endpoint via environment variable override

### Documentation

- [x] **DOCS-01**: README provides one-command quickstart with prerequisites, `HF_TOKEN` setup steps, and explicit cold-start warning
- [x] **DOCS-02**: Cost breakdown documents scale-to-zero vs always-on trade-offs and monthly math
- [x] **DOCS-03**: NVIDIA-recommended sampling defaults (`--temp 1.0 --top-p 0.95`) are documented with source reference
- [x] **DOCS-04**: EU-RO-1 datacenter restriction is noted with latency implications for non-EU users
- [x] **DOCS-05**: Context window tuning documents `-c` flag options and VRAM/RAM constraints
- [x] **DOCS-06**: MIT LICENSE file is included in the repository

### Model Caching

- [~] **CACHE-01**: RunPod cached model used on cold start — *Deferred: RunPod cached models downloads all repo files (2.01 TB); no selective quant filtering. Network volume approach used instead.*
- [~] **CACHE-02**: Worker resolves cached model path dynamically — *Deferred: `get_cached_model_path()` exists but dormant (`CACHED_REPO_ID=""`)*
- [~] **CACHE-03**: Only UD-Q4_K_XL quantization cached — *Deferred: same reason as CACHE-01*
- [x] **CACHE-04**: Network volume repurposed for binary cache only (not model storage) — *Partially met: volume stores both the model AND the cached llama-server binary. Model download still happens in seed, but cold start is fast because model is already on volume.*

### Streaming

- [x] **STRM-01**: Flash LB SSE pass-through confirmed working — FastAPI `StreamingResponse` passes through unchanged
- [x] **STRM-02**: `/v1/chat/completions` with `stream: true` returns `text/event-stream` SSE chunks
- [~] **STRM-03**: OpenCode + Open WebUI verified live. Claude Code and Mistral Vibe untested against live endpoint — *implementation is correct; partial credit*

## v2 Requirements

- **ENH-01**: Streaming config examples in integration snippets — carried to v0.2.0
- **ENH-02**: Other quant variants (Q8_0 for quality, Q3_K_M for smaller VRAM)
- **ENH-03**: Multi-region deployment guide (pending RunPod Flash expansion beyond EU-RO-1)
- **ENH-04**: GitHub Actions deploy workflow for automated re-deployment

## Out of Scope

| Feature | Reason |
|---------|--------|
| vLLM FP8 / 2×H100 | Requires 2×H100, exceeds single-user budget |
| Fine-tuning / model training | Inference only |
| Multi-user / team deployment | Serverless cold starts incompatible with shared use |
| Web UI / chat interface | Pure API backend |
| Other model support | Nemotron-3-Super-120B only |
| Streaming enabled by default | Keep deployment script simple; documented as optional |
